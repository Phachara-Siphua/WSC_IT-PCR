บทนำ
ใน session นี้ คุณจะทำหน้าที่เป็นนักพัฒนาแอปพลิเคชันมือถือสำหรับลูกค้าของ Belle Croissant Lyonnais แอปนี้จะช่วยให้ลูกค้าสามารถเข้าดูสินค้า สั่งซื้อ และติดตามสถานะการสั่งซื้อ โดยเป้าหมายหลักของ session นี้คือให้คุณสร้างแอปที่ใช้งานง่าย มีหน้าตาสวยงาม และสามารถเชื่อมต่อกับระบบที่พัฒนาขึ้นใน session 3 ได้อย่างราบรื่น

ทักษะที่จะได้รับการทดสอบ:
การพัฒนาแอปพลิเคชันมือถือ: การสร้างแอปที่ใช้งานได้ดีบนอุปกรณ์มือถือ
การเชื่อมต่อ API: การเชื่อมต่อแอปกับระบบที่มีอยู่ผ่าน API
การออกแบบอินเทอร์เฟซ: การสร้างหน้าตาแอปที่ใช้งานง่ายและดูดี
การแก้ปัญหา: การค้นหาและแก้ไขปัญหาที่เกิดขึ้นระหว่างการพัฒนาแอป

เนื้อหาของ session
session นี้ประกอบด้วยเนื้อหาดังนี้:
คู่มือการทำงาน (Session Instructions): เป็นเอกสาร PDF ที่มีรายละเอียดเกี่ยวกับงานที่ต้องทำและสิ่งที่ต้องส่ง
โฟลเดอร์ข้อมูลทั่วไป (Common Folder): โฟลเดอร์นี้มีทรัพยากรต่างๆ เช่น โลโก้ Belle Croissant Lyonnais, ไอคอน, คู่มือการออกแบบ และทรัพยากรอื่นๆ ที่ใช้ในการพัฒนาแอป
เอกสาร API (API Documentation): คู่มือเกี่ยวกับ API ของผลิตภัณฑ์, ลูกค้า, และการสั่งซื้อ ซึ่งให้ข้อมูลเกี่ยวกับ endpoint, รูปแบบการส่งข้อมูล/ตอบกลับ และการยืนยันตัวตน

คำอธิบายโปรเจกต์และงานที่ต้องทำ
ใน session นี้ คุณจะต้องพัฒนาแอป Belle Croissant Lyonnais โดยเพิ่มคุณสมบัติใหม่ๆ สำหรับการสั่งซื้อแบบกำหนดเอง, การจัดการผู้ใช้, และการประมวลผลคำสั่งซื้อ

แนวทางการพัฒนา:
1.ใช้งานง่าย: ทำให้แอปใช้งานง่ายและเป็นมิตรกับผู้ใช้
2.หน้าตาดี: ปฏิบัติตามคู่มือการออกแบบของ Belle Croissant Lyonnais
3.ทำงานได้ดี: ตรวจสอบให้แน่ใจว่าทุกส่วนของแอปทำงานได้อย่างถูกต้องและไม่มีข้อผิดพลาด
4.ปลอดภัย: ปกป้องข้อมูลผู้ใช้และทำให้แอปปลอดภัยในการใช้งาน
5.ตรงเวลา: ทำงานให้เสร็จภายในเวลาที่กำหนด

ข้อควรพิจารณาทางเทคนิค:
1.การเชื่อมต่อ API: ปฏิบัติตามแนวทางการออกแบบ Web API
2.การตรวจสอบข้อมูล (Data Validation): ตรวจสอบความถูกต้องของข้อมูลผู้ใช้ก่อนส่งไปยัง API
3.การจัดการข้อผิดพลาด: แสดงข้อความที่ชัดเจนให้กับผู้ใช้หากเกิดปัญหา

ข้อแนะนำในการพัฒนาแอปพลิเคชัน
4.1 การออกแบบฐานข้อมูลสำหรับการจัดการผู้ใช้
วัตถุประสงค์: ออกแบบฐานข้อมูลที่เก็บข้อมูลผู้ใช้และจัดการข้อมูลเกี่ยวกับการยืนยันตัวตน ข้อมูลโปรไฟล์ ที่อยู่ และการสมัครรับจดหมายข่าว

งานที่ต้องทำ: วิเคราะห์ข้อมูลตัวอย่างผู้ใช้และสร้าง แผนภาพเอนทิตี-ความสัมพันธ์ (Entity-Relationship Diagram หรือ ERD) ที่รองรับการเก็บข้อมูลนี้รวมถึงการขยายข้อมูลในอนาคต

สิ่งที่ต้องส่ง:
ไฟล์ PDF ที่ประกอบด้วย ERD ของฐานข้อมูลที่ออกแบบ

หมายเหตุเพิ่มเติม:
ฐานข้อมูลต้องเก็บที่อยู่ผู้ใช้ได้หลายแห่ง และสามารถระบุที่อยู่หลักได้
เก็บข้อมูลการสมัครรับจดหมายข่าวของผู้ใช้
เก็บรหัสผ่านและคำตอบของคำถามเพื่อความปลอดภัยอย่างปลอดภัย
เก็บ URL ของรูปโปรไฟล์ผู้ใช้
รองรับการเพิ่มหรือแก้ไขโครงสร้างข้อมูลในอนาคต

4.2 การพัฒนา API สำหรับการจัดการผู้ใช้
วัตถุประสงค์: สร้าง API สำหรับการจัดการผู้ใช้ Belle Croissant Lyonnais ตามโครงสร้างฐานข้อมูลที่ออกแบบในงาน 4.1 และเชื่อมต่อกับ API สำหรับผลิตภัณฑ์และการสั่งซื้อหลัก

งานที่ต้องทำ:
สร้าง endpoint สำหรับ API เช่น การลงทะเบียน การเข้าสู่ระบบ การตั้งค่ารหัสผ่านใหม่ การอัปเดตโปรไฟล์ การจัดการที่อยู่ และการสมัคร/ยกเลิกการสมัครจดหมายข่าว
สิ่งที่ต้องส่ง:
รายชื่อ endpoint ของ API พร้อมตัวอย่าง request/response ในรูปแบบ JSON

4.3 การออกแบบหน้าจอลงทะเบียนผู้ใช้
วัตถุประสงค์: ออกแบบและสร้างหน้าจอที่ให้ผู้ใช้ลงทะเบียนและสมัครจดหมายข่าวได้

งานที่ต้องทำ:
สร้าง UI ตามที่ระบุใน wireframe รวมถึงการตรวจสอบข้อมูลก่อนส่งไปยัง API
ส่งคำขอลงทะเบียนไปยัง API และจัดการข้อผิดพลาดหากการลงทะเบียนไม่สำเร็จ

สิ่งที่ต้องส่ง:
รวมหน้าจอลงทะเบียนเข้าไปในแอป Belle Croissant Lyonnais

4.4 การออกแบบหน้าจอเข้าสู่ระบบ
วัตถุประสงค์: ออกแบบหน้าจอเข้าสู่ระบบที่ใช้งานง่ายและเชื่อมต่อกับ API การจัดการผู้ใช้อย่างปลอดภัย

งานที่ต้องทำ:
สร้างหน้าจอเข้าสู่ระบบตาม wireframe ที่กำหนด รวมถึงการตรวจสอบข้อมูลและการจัดการข้อผิดพลาด

สิ่งที่ต้องส่ง:
รวมหน้าจอเข้าสู่ระบบเข้าไปในแอป
สร้างบัญชีผู้ใช้ทดสอบที่มีข้อมูลที่กำหนด

4.5 การออกแบบหน้าจอลืมรหัสผ่าน
วัตถุประสงค์: ออกแบบหน้าจอ "ลืมรหัสผ่าน" เพื่อให้ผู้ใช้สามารถตั้งรหัสผ่านใหม่ได้
งานที่ต้องทำ:

สร้างหน้าจอสำหรับลืมรหัสผ่านตามที่ระบุ รวมถึงการแสดงคำถามเพื่อความปลอดภัยหลังจากที่ผู้ใช้กรอกอีเมลเรียบร้อย
ส่งคำขอตั้งรหัสผ่านใหม่ไปยัง API และจัดการข้อผิดพลาดที่อาจเกิดขึ้น

สิ่งที่ต้องส่ง:
รวมหน้าจอลืมรหัสผ่านเข้าไปในแอป
สร้างบัญชีผู้ใช้ทดสอบสำหรับทดสอบฟีเจอร์นี้

4.6 การออกแบบหน้าจอโปรไฟล์ผู้ใช้
วัตถุประสงค์: ออกแบบหน้าจอโปรไฟล์เพื่อให้ผู้ใช้สามารถดูและแก้ไขข้อมูลส่วนตัว จัดการที่อยู่ และดูประวัติการสั่งซื้อ

งานที่ต้องทำ:
สร้าง UI สำหรับการแสดงและแก้ไขข้อมูลส่วนตัวของผู้ใช้ รวมถึงการจัดการที่อยู่ และการแสดงประวัติการสั่งซื้อ
อนุญาตให้ผู้ใช้แก้ไขข้อมูลได้บนหน้าจอและแสดงข้อความยืนยันเมื่อข้อมูลถูกบันทึกสำเร็จ

สิ่งที่ต้องส่ง:
รวมหน้าจอโปรไฟล์ การจัดการที่อยู่ และประวัติการสั่งซื้อเข้าไปในแอป
